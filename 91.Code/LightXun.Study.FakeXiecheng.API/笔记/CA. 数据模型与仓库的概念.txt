1. 模型的作用
1) 映射数据库, 对象化数据
2) 获取数据, 更新数据, 传递数据, 保存数据等
3) 处理业务逻辑, 视为 BusinessLayer 业务层
4) View Model 与视图数据绑定
5) 定义不明确, 经验很重要
6) 可以是简单的 POCO 类
7) 也可以是复杂的充血性领域模型(Rich Domain Model)
8) 模型中不仅要包含属性, 也要包含业务逻辑

2. 获取数据的方式
1) JDBC
2) Ado.Net
3) ORM
4) repository 仓库模式

3. repository 仓库模式 - 目前主流的数据持久化的设计模式
1) 可以帮助程序员使用对象化的模式, 不用去关心数据是如何保存的, 甚至可以忽略数据库的类型

4. 使用模型和数据仓库的好处
1) 业务逻辑与数据模型紧密耦合, 减少分层, 降低代码数量
2) 完全剥离数据库业务, 程序员可以更专注于实现业务逻辑
3) 面向对象编程, 数据转化为对象, 更符合程序员的思考方式
