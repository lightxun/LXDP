1. HTTP 协议规定了不同方法的安全特性以及幂等特性, 后端服务必须为前端提供这些特性

2. 安全性(通常指取值)
1) 意味着一个操作用于获取信息而非修改信息, 而且该操作无论经过多少次调用都不会产生副作用, 不会更改当前资源的状态
2) 安全操作包括哪些可以被缓存、对资源无损预加载的方法(比如 GET 请求、HEAD 请求)
3）错误的表述: GET api/product/1234/delete HTTP/1.1
4) 对于不安全的请求, 要进行数据验证, 确保可以较为安全的进行操作和处理

3. 幂等性(通常指赋值)
1) 意味着同样的操作不管经过多少次调用, 返回的数据、或者产生的效果都是一致的
2) 幂等性是数学概念, 表达的是 N 次变换与 1 次变换的结果相同

4. HTTP 方法的安全性和幂等性
1) GET: 是安全性, 是幂等性, 用于获取资源信息, 并返回主体
2) HEAD: 是安全性, 是幂等性, 用于只请求头部信息
3) OPTION: 是安全性, 是幂等性, 用于查看服务器信息
4) DELETE: 不是安全性, 是幂等性, 用于删除资源, 由于无论调用多少次删除, 都只会删除那指定的一个, 所以是幂等的
5) PUT: 不是安全性, 是幂等性, 用于更新资源, 由于同一个请求无论请求多少次, 都只会更新那指定的一个, 所以是幂等的, 而 PUT 请求一定会改变数据资源, 所以是不安全的
6) POST: 不是安全性, 不是幂等性, 用于创建资源, 由于调用的每一次, 都会新增一条数据, 会改变资源状态
