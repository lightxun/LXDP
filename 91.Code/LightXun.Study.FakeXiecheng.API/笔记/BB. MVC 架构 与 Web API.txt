1. 什么是 MVC
1) MVC 是软件工程的架构方式
2) 模型(Model) 视图(View) 控制器(Controller)
3) 可以轻松分离业务操作 数据显示 逻辑控制
4) 视图 View 是用户交互界面, 仅展示数据 不处理数据, 接收用户输入
5) 模型 Model 是 MVC 架构核心, 表示业务模型或数据模型, 业务逻辑 如算法实现 数据的管理 输出对象的封装等等
6) 控制器 Controller 是接受用户的输入, 并调用模型和视图去完成用户的请求处理, 不处理数据, 仅接受用户的请求, 并决定调用那个模型组件去处理数据, 根据模型的返回, 决定使用那个视图来显示数据

2. 三层架构
1) UI层, 表示用户界面
2) BLL(Business Logic Layer) 业务逻辑层, 处理核心业务以及数据封装
3) DAL(Data Access Layer) 数据链路层, 表示数据访问

3. MVC VS 三层架构
1) 三层架构面向接口编程, 而三个层级之间的完全解耦、完全可替换
2) MVC 的每个部分都是紧密结合的, 它的核心并不是解耦, 而是重用
3) 三层架构是描述一个自下而上, 具有明显层级的架构
4) MVC架构是水平的, 只有调用关系, 没有层级关系, 所有数据的流动都是通过数据绑定和事件驱动处理的

4. MVC 的优点
1) 耦合性低
2) 可复用性高
3) 可维护性高

5. MVC 的缺点
1) 定义不明确, 学习曲线陡
2) 结构复杂
3) 数据流动效率低
